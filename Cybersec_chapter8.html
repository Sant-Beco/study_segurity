// sección 01 — arsenal OSINT

Las herramientas que
los atacantes usan primero

Todas son legales, públicas y gratuitas. Lo que las hace peligrosas es cómo se combinan. Úsalas en tus propios sistemas para ver qué ves desde fuera.

PASIVO Reconocimiento
theHarvester
Agrega emails, subdominios, IPs y nombres de personas desde Google, Bing, LinkedIn, Shodan y más. Una sola herramienta, muchas fuentes.
$ theHarvester -d tudominio.com -l 200 -b all
PASIVO Red
Shodan
Motor de búsqueda de dispositivos conectados. Ve qué puertos, servicios y versiones expone tu servidor según el índice de Shodan. También muestra CVEs.
$ shodan host TU_IP_PUBLICA
$ shodan search "org:tuempresa nginx"
PASIVO Certificados
crt.sh
Certificate Transparency Logs. Cada certificado TLS emitido queda registrado públicamente. Revela subdominios históricos incluyendo staging, admin, dev.
$ curl "https://crt.sh/?q=%.tudominio.com&output=json"
PASIVO DNS / WHOIS
dnsx / WHOIS
Registros DNS completos: A, MX, TXT, SPF, DKIM, NS. WHOIS revela registrante, fechas y nameservers. Mapeo completo de la infraestructura.
$ dnsx -d tudominio.com -a -mx -txt -resp
$ whois tudominio.com
CON CUIDADO Social
Maltego
Grafo visual de relaciones entre personas, dominios, IPs, emails y redes sociales. Visualiza cómo está conectada la infraestructura de un objetivo.
$ maltego # GUI
# Transforma: dominio → emails → personas → LinkedIn
PASIVO Metadatos
ExifTool
Extrae metadatos de imágenes, PDFs y documentos. GPS, cámara, autor, empresa, software. Una foto subida sin limpiar puede revelar la ubicación exacta.
$ exiftool foto.jpg
$ exiftool -all= foto.jpg # limpiar
PASIVO Archivos
Google Dorks
Búsquedas avanzadas en Google que revelan archivos sensibles indexados, directorios expuestos, configuraciones, y paneles de admin accesibles.
site:tudominio.com filetype:env
site:tudominio.com inurl:admin
PASIVO Historial
Wayback Machine
Archivos históricos del sitio web. Versiones antiguas pueden exponer endpoints eliminados, credenciales commiteadas, o estructuras de API que ya no están en producción.
$ waybackurls tudominio.com | grep -E "\.env|\.sql|backup"
PASIVO Código
GitHub Dorks
Búsqueda en código público de GitHub. Encuentra credenciales, API keys, connection strings y secrets commiteados accidentalmente por desarrolladores.
org:tuempresa "SECRET_KEY"
org:tuempresa "DATABASE_URL" password

// sección 03 — metodología

Cómo un atacante
reconoce tu infraestructura

Este es el proceso exacto — paso a paso — antes de que haya un solo ataque activo. Todo pasivo, todo legal, todo público.

01
Dominio
WHOIS + DNS completo. Registrante, nameservers, fechas de registro, registros MX, SPF, DKIM. Revela el proveedor de hosting, el proveedor de email, y si hay protección de privacidad en el WHOIS. whois tudominio.com && dnsx -d tudominio.com -all
Pasivo
02
Subdominios
Certificate Transparency + amass pasivo. crt.sh revela todos los certificados TLS emitidos históricamente. Cada subdominio es un potencial punto de entrada: staging, admin, dev, api-v2, internal. curl "https://crt.sh/?q=%.tudominio.com&output=json"
Pasivo
03
Personas
theHarvester + LinkedIn. Emails corporativos, nombres completos, cargos y departamentos. Esto alimenta ataques de phishing dirigido (spear phishing) y permite adivinar patrones de email para encontrar más cuentas. Los developers con código en GitHub son especialmente valiosos. theHarvester -d tudominio.com -b linkedin,google
Pasivo
04
Puertos
Shodan. Sin enviar un solo paquete al servidor: qué puertos están abiertos, qué servicios corren, qué versiones exactas, qué CVEs tienen. Si 5432 aparece: PostgreSQL público. Si 8000 aparece: Uvicorn sin proxy. Si hay CVEs: vector de entrada inmediato. shodan host TU_IP
Pasivo
05
Código
GitHub dorks + GitLeaks. Búsqueda en repositorios públicos de la organización. Un commit hace 6 meses con SECRET_KEY en texto plano, aunque ya fue eliminado, puede seguir en el historial de Git. git log --all -S "SECRET_KEY" lo encontraría. gitleaks detect --source . --log-opts="--all"
Pasivo
06
Análisis
Consolidar y priorizar vectores. Con toda la información recopilada, el atacante tiene: emails para phishing, subdominios para probar, IPs con CVEs conocidos, tecnologías exactas, y posiblemente credenciales. El reconocimiento termina — el ataque empieza.
Análisis

// sección 06 — forense digital

Forense digital:
investigar después del incidente

El forense digital responde «qué pasó exactamente». No eliminar evidencia. Documentar todo. Preservar la cadena de custodia.

01
📄
Preservación de Evidencia
Lo primero: no tocar el sistema comprometido más de lo necesario. Hacer una imagen forense del disco antes de cualquier acción. Los logs son evidencia — nunca borrarlos durante un incidente activo.
→ dd, dcfldd, FTK Imager
→ docker commit cipher_app cipher_forensic:$(date +%s)
02
📋
Análisis de Logs
Extraer el período relevante, buscar patrones de ataque, identificar la IP origen, el endpoint afectado, y la secuencia exacta de acciones. Los logs estructurados JSON facilitan el análisis.
→ jq, grep, awk
→ Splunk, ELK Stack, Loki
03
🔎
Timeline del Ataque
Reconstruir la secuencia temporal exacta: primer acceso, lateralizó, escaló privilegios, exfiltró datos. El timestamp de cada log, acceso a DB y cambio de archivo forma el mapa del ataque.
→ Volatility (memoria), Autopsy (disco)
→ plaso / log2timeline
04
🌐
Análisis de Red
Si tienes capturas de red del período, Wireshark permite ver exactamente qué datos viajaron. Identificar exfiltración de datos, conexiones a C&C, y volumen transferido.
→ Wireshark, tcpdump
→ NetworkMiner, Zeek
05
📊
Evaluación del Impacto
¿Qué datos fueron accedidos? ¿Modificados? ¿Exfiltrados? Consultas SQL en el período afectado, archivos con timestamps modificados, usuarios cuyas contraseñas pueden estar comprometidas.
→ PostgreSQL: pg_stat_statements, pg_audit
→ SIEM: correlación de eventos
06
📝
Cadena de Custodia
Documentar quién accedió a la evidencia, cuándo y para qué. Hash SHA-256 de todos los archivos de evidencia antes y después. Esencial si el incidente va a procesos legales.
→ sha256sum evidencia.tar.gz > hashes.txt
→ Firmar con GPG

# app/tools/osint_recon.py — OSINT pasivo con Python
# Solo para uso en sistemas propios o con permiso escrito
import asyncio, httpx, whois, dns.resolver
from urllib.parse import urlparse
from loguru import logger
from dataclasses import dataclass
from typing import Optional
 
@dataclass
class DomainRecon:
    domain: str
    registrar: Optional[str] = None
    creation_date: Optional[str] = None
    name_servers: list[str] = None
    mx_records: list[str] = None
    subdomains: list[str] = None
    open_ports: list[int] = None
 
async def recon_domain(domain: str) -> DomainRecon:
    result = DomainRecon(domain=domain)
 
    # 1. WHOIS — información del registrante
    try:
        w = whois.whois(domain)
        result.registrar = w.registrar
        result.creation_date = str(w.creation_date)
        result.name_servers = w.name_servers or []
        logger.info("whois_complete", domain=domain, registrar=result.registrar)
    except Exception as exc:
        logger.warning("whois_failed", domain=domain, error=str(exc))
 
    # 2. DNS — registros MX, A, TXT, SPF
    result.mx_records = []
    for rtype in ("MX", "A", "TXT", "NS"):
        try:
            answers = dns.resolver.resolve(domain, rtype)
            result.mx_records.extend([str(r) for r in answers])
        except: pass
 
    # 3. Subdomain enumeration (pasivo — fuentes públicas)
    result.subdomains = await enumerate_subdomains_passive(domain)
 
    return result
 
async def enumerate_subdomains_passive(domain: str) -> list[str]:
    # Certificate Transparency Logs — completamente pasivo
    found = set()
    url = f"https://crt.sh/?q=%.{domain}&output=json"
    async with httpx.AsyncClient(timeout=15) as client:
        try:
            r = await client.get(url)
            for entry in r.json():
                name = entry.get("name_value", "")
                for sub in name.split("\n"):
                    if sub.endswith(f".{domain}"):
                        found.add(sub.strip())
        except: pass
    return sorted(found)

from PIL import Image
import io, json
 
# ── IMÁGENES: extraer y limpiar EXIF
def extract_image_metadata(image_bytes: bytes) -> dict:
    img = Image.open(io.BytesIO(image_bytes))
    exif_raw = img.getexif()
 
    # Tags de interés para OSINT
    TAGS_OSINT = {
        271: "camera_make",
        272: "camera_model",
        306: "datetime_original",
        34853: "gps_data",  # ← coordenadas GPS!
        37510: "user_comment",
        40091: "author",
        40094: "keywords",
    }
    metadata = {}
    for tag_id, name in TAGS_OSINT.items():
        val = exif_raw.get(tag_id)
        if val: metadata[name] = str(val)
    return metadata
 
def strip_image_metadata(image_bytes: bytes) -> bytes:
    # Abrir y re-guardar elimina TODO el EXIF
    img = Image.open(io.BytesIO(image_bytes))
    img = img.convert("RGB")  # elimina canales auxiliares
    output = io.BytesIO()
    img.save(output, format="JPEG", quality=85)
    # Resultado: sin EXIF, sin GPS, sin cámara
    return output.getvalue()
 
# ── PDFs: extraer y limpiar metadatos
import pikepdf
 
def extract_pdf_metadata(pdf_path: str) -> dict:
    with pikepdf.open(pdf_path) as pdf:
        meta = pdf.docinfo
        return {
            "author": str(meta.get("/Author", "")),
            "creator": str(meta.get("/Creator", "")),
            "created": str(meta.get("/CreationDate", "")),
            "producer": str(meta.get("/Producer", "")),
        }
 
def strip_pdf_metadata(pdf_path: str, output_path: str):
    with pikepdf.open(pdf_path) as pdf:
        # Eliminar todos los metadatos del documento
        del pdf.docinfo
        pdf.save(output_path)
    # PDF limpio: sin autor, sin empresa, sin software

# Requiere cuenta en shodan.io (plan gratuito disponible)
 
import shodan, os
from loguru import logger
 
SHODAN_API_KEY = os.getenv("SHODAN_API_KEY")
 
def scan_my_exposure(ip: str) -> dict:
    # Busca QUÉ ve Shodan sobre tu IP pública
    # Completamente pasivo — solo consulta la base de datos de Shodan
    api = shodan.Shodan(SHODAN_API_KEY)
    try:
        host = api.host(ip)
        exposure = {
            "ip": host["ip_str"],
            "org": host.get("org"),
            "os": host.get("os"),
            "ports": host["ports"],
            "vulns": host.get("vulns", []),
            "hostnames": host.get("hostnames", []),
            "last_update": host["last_update"],
        }
 
        # Alertar si hay puertos peligrosos expuestos
        DANGEROUS_PORTS = {22, 3306, 5432, 6379, 27017, 9200}
        exposed_dangerous = DANGEROUS_PORTS & set(exposure["ports"])
        if exposed_dangerous:
            logger.error("dangerous_ports_exposed",
                ports=list(exposed_dangerous),
                ip=ip
            )
 
        # Alertar si hay CVEs conocidos
        if exposure["vulns"]:
            logger.error("cves_found_on_ip",
                cves=exposure["vulns"],
                ip=ip
            )
 
        return exposure
    except shodan.APIError as exc:
        logger.warning("shodan_api_error", error=str(exc))
        return {}

// sección 08 — límites éticos y legales

La línea entre
investigación y delito

El mismo conocimiento, diferente aplicación. Entender el límite no es opcional — cruzarlo es un delito en prácticamente todos los países.

✓ Legal — fuentes públicas con propósito legítimo
✓Buscar información sobre tu propia infraestructura en Shodan
✓Usar crt.sh para ver tus propios subdominios expuestos
✓Google dorks en tu propio dominio para auditoría
✓WHOIS y DNS lookup de cualquier dominio (dato público)
✓Buscar en GitHub repositorios públicos de tu organización
✓Wayback Machine en cualquier URL pública
✓Bug bounty en programas autorizados con scope definido
✓Pentest con contrato escrito y scope firmado
✗ Ilegal — sin permiso explícito por escrito
✗Escanear puertos de sistemas que no te pertenecen (incluso pasivo)
✗Acceder a subdominios de otros aunque estén sin autenticación
✗Descargar archivos de configuración expuestos accidentalmente
✗Usar credenciales encontradas aunque sean de un breach antiguo
✗Reportar vulnerabilidades sin seguir el proceso de responsible disclosure
✗Hacer web scraping masivo en sistemas de terceros
✗Almacenar o distribuir datos personales encontrados
✗Cualquier explotación activa sin contrato firmado
⚠ NOTA LEGAL: La ley de delitos informáticos varía por país. En España (Código Penal art. 197bis), Colombia (Ley 1273), México (CPEUM art. 211bis) y la mayoría de países latinoamericanos, el acceso no autorizado a sistemas es delito aunque el sistema no tenga contraseña. El hecho de que algo sea accesible técnicamente no lo hace legal.
