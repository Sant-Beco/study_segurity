CAPA 7 — APLICACIÓN
Application
Aquí vive tu aplicación FastAPI. Esta capa define cómo se estructura y presenta la información al usuario final. HTTP, el protocolo que une internet, opera en esta capa. Cuando defines un endpoint @app.get("/users"), estás trabajando enteramente en capa 7.

Protocolos en esta capa

HTTP/1.1 HTTP/2 HTTP/3 HTTPS DNS SMTP WebSocket
→ FastAPI opera aquí. Cada @router, cada modelo Pydantic,
cada response JSON vive en esta capa.

// sección 04 — conexión con tu stack

Cómo FastAPI
vive en las capas

Cada request que llega a tu API recorre exactamente este camino. Entender esto es entender dónde poner tus defensas.

Cliente
Navegador / App / curl Genera el HTTP Request. Puede ser un humano o un atacante. CAPA 7
↓ TCP Connection :443 (TLS 1.3)
Proxy
Nginx / Caddy Termina TLS. Balancea carga. Primera línea de defensa contra DoS. CAPA 7 / 4
↓ HTTP (interno) → localhost:8000
ASGI Server
Uvicorn Servidor ASGI asincrónico. Escucha el socket TCP, parsea HTTP, pasa al framework. ASGI / HTTP
↓ ASGI scope, receive, send
Framework
FastAPI Middleware chain → Router → Dependency Injection → Pydantic validation → Handler. Tu código
↓ SQLAlchemy async session
Base de Datos
PostgreSQL 15 Datos persistidos. Conexión por socket local o TCP cifrado. Alembic gestiona el esquema. TCP :5432

Tu primera API segura

Ejecutar
# main.py — punto de entrada de tu FastAPI from fastapi import FastAPI from fastapi.middleware.cors import CORSMiddleware from fastapi.middleware.trustedhost import TrustedHostMiddleware from app.routers import users, auth from app.middleware.security import SecurityHeadersMiddleware app = FastAPI( title="CipherDoc API", version="0.1.0", docs_url="/docs", # desactivar en producción ) # ── Middleware de seguridad (orden importa) app.add_middleware(TrustedHostMiddleware, allowed_hosts=["*"]) app.add_middleware(SecurityHeadersMiddleware) app.add_middleware( CORSMiddleware, allow_origins=["https://tudominio.com"], allow_credentials=True, allow_methods=["GET", "POST", "PUT", "DELETE"], allow_headers=["*"], ) # ── Routers app.include_router(users.router, prefix="/api/v1") app.include_router(auth.router, prefix="/api/v1") @app.get("/") async def root(): return {"status": "online", "version": "0.1.0"}
# routers/users.py — endpoints de usuarios from fastapi import APIRouter, Depends, HTTPException, status from sqlalchemy.ext.asyncio import AsyncSession from app.db import get_db from app.schemas import UserCreate, UserResponse from app.services import user_service from app.auth import get_current_user router = APIRouter(prefix="/users", tags=["users"]) @router.get("/", response_model=list[UserResponse]) async def list_users( db: AsyncSession = Depends(get_db), current_user = Depends(get_current_user), # auth requerida skip: int = 0, limit: int = 100, ): """Listar usuarios. Solo admins.""" if not current_user.is_admin: raise HTTPException( status_code=status.HTTP_403_FORBIDDEN, detail="Acceso denegado" ) return await user_service.get_users(db, skip=skip, limit=limit) @router.post("/", response_model=UserResponse, status_code=201) async def create_user( user_data: UserCreate, # Pydantic valida automáticamente db: AsyncSession = Depends(get_db), ): # Verificar duplicados existing = await user_service.get_by_email(db, user_data.email) if existing: raise HTTPException(status_code=409, detail="Email ya registrado") return await user_service.create_user(db, user_data)
# middleware/security.py — headers de seguridad HTTP from starlette.middleware.base import BaseHTTPMiddleware from starlette.requests import Request class SecurityHeadersMiddleware(BaseHTTPMiddleware): async def dispatch(self, request: Request, call_next): response = await call_next(request) # Previene clickjacking response.headers["X-Frame-Options"] = "DENY" # Previene MIME sniffing response.headers["X-Content-Type-Options"] = "nosniff" # Fuerza HTTPS (HSTS) response.headers["Strict-Transport-Security"] = ( "max-age=31536000; includeSubDomains" ) # Content Security Policy response.headers["Content-Security-Policy"] = ( "default-src 'self'; script-src 'self'" ) # Ocultar información del servidor response.headers["X-Powered-By"] = "" response.headers["Server"] = "" return response

Protocolos que definen
la seguridad en tránsito

TLS
Usar siempre

TLS 1.3

Transport Layer Security. Cifra toda la comunicación. Versión 1.3 eliminó algoritmos débiles. Es lo que convierte HTTP en HTTPS. Tu Uvicorn debe estar detrás de Nginx con TLS.

→ Nginx + Certbot (Let's Encrypt gratis)
JWT
Con cuidado

JWT / OAuth 2.0

JSON Web Tokens para autenticación stateless. Peligroso si se mal-implementa: algoritmo 'none', claves débiles, o sin expiración. FastAPI tiene soporte nativo vía python-jose.

→ python-jose + passlib[bcrypt]
SSH
Usar siempre

SSH

Secure Shell. Acceso remoto cifrado a tus servidores. Nunca exponer contraseñas por SSH — siempre usar claves RSA/ED25519. Tu deploy en DigitalOcean/Render depende de esto.

→ ssh-keygen -t ed25519
SSL
Obsoleto

SSL / TLS 1.0-1.1

Protocolos viejos con vulnerabilidades conocidas (POODLE, BEAST, DROWN). Nunca habilitar en producción. Los scanners de seguridad los detectarán inmediatamente.

→ Deshabilitar en nginx.conf
VPN
Recomendado

WireGuard / VPN

Para acceso seguro a tu base de datos PostgreSQL en producción. Nunca exponer el puerto 5432 al internet público. WireGuard es más rápido y seguro que OpenVPN.

→ Puerto 5432 solo accesible internamente
DNS
Punto débil

DNS & DNSSEC

El DNS es el directorio del internet. Sin DNSSEC, un atacante puede redirigir tu dominio a su servidor (DNS Spoofing). Cloudflare ofrece protección DNS gratuita.

→ Cloudflare DNS + DNSSEC activado

Ataques de red que
debes conocer

⬛
Man-in-the-Middle (MITM)Intercepción de tráfico
El atacante se posiciona entre cliente y servidor para interceptar y modificar el tráfico. Sin TLS, cualquier dato en tránsito es legible. Común en redes Wi-Fi públicas.
9.8
⬛
SYN Flood / DDoSDenegación de servicio
Saturar tu servidor con requests TCP a medio completar. Agota los recursos del sistema. Tu FastAPI caerá si no tienes rate limiting y un proxy delante.
9.1
⬛
DNS SpoofingEnvenenamiento de caché DNS
Redirigir el dominio de tu API a un servidor malicioso sin que el usuario lo sepa. DNSSEC y HTTPS con certificados verificados son las defensas principales.
8.1
⬛
HTTP Header InjectionManipulación de headers
Inyectar headers maliciosos en requests o responses. Tu middleware de seguridad en FastAPI debe sanitizar y establecer headers seguros en cada response.
7.5
⬛
SSL StrippingDowngrade de HTTPS a HTTP
Forzar al cliente a usar HTTP en lugar de HTTPS. El header HSTS (Strict-Transport-Security) que implementaste en tu middleware es la defensa directa.
6.8
ENTRADA #002 — CAPÍTULO 02 COMPLETADO

El modelo OSI deja de ser teoría abstracta cuando lo ves directamente en tu stack: cada request de FastAPI comienza en la capa 7 (Aplicación), pasa por Uvicorn en la capa de transporte TCP, viaja cifrado por TLS, y llega físicamente a través de cables o radio. Cada capa es un punto donde algo puede salir mal — o donde puedo defender.

Lo más importante de este capítulo: los headers de seguridad HTTP no son opcionales. Ese middleware que escribí hoy — X-Frame-Options, HSTS, CSP — son la primera línea de defensa real en mi código, antes de que llegue a cualquier lógica de negocio.
