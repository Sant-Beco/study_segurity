OWASP Top 10
2021 — Las vulnerabilidades

El estándar global. Cada empresa seria las conoce. Cada una tiene su versión en código Python real. Click para expandir cada vulnerabilidad.

A01
Broken Access Control
Control de acceso roto — #1 desde 2021
CRÍTICO

Un usuario puede acceder a recursos que no le pertenecen. El ejemplo más común: /api/users/42 devuelve el perfil del usuario 42 sin verificar si quien pregunta ES el usuario 42.

En FastAPI esto se llama IDOR (Insecure Direct Object Reference). Cambias el ID en la URL y ves datos de otro usuario.

Cómo lo detecta un attacker:
GET /api/users/1 → OK
GET /api/users/2 → también OK? 🚨

# ❌ VULNERABLE @router.get("/users/{user_id}") async def get_user(user_id: int, db=Depends(get_db)): return await db.get(User, user_id) # sin auth! # ✅ SEGURO @router.get("/users/{user_id}") async def get_user( user_id: int, current_user = Depends(get_current_user), db = Depends(get_db) ): if current_user.id != user_id and not current_user.is_admin: raise HTTPException(403, "Acceso denegado") return await db.get(User, user_id)
A02
Cryptographic Failures
Datos sensibles en texto plano o cifrado débil
CRÍTICO

Guardar contraseñas con MD5 o SHA-1, transmitir datos sensibles sin TLS, usar claves hardcodeadas en el código. Vimos esto en el capítulo anterior, pero aquí lo vemos desde el ángulo del atacante.

Si roban tu DB y las contraseñas son MD5:
Una GPU rompe ~10 billones hashes/seg
Con bcrypt: ~4,000/seg → 2.5 millones x más seguro

# ❌ VULNERABLE import hashlib password_hash = hashlib.md5(password.encode()).hexdigest() # También peligroso: SHA1, SHA256 sin salt # ❌ VULNERABLE — hardcoded secret SECRET_KEY = "mysecretkey" # en el código! # ✅ SEGURO from passlib.context import CryptContext pwd_context = CryptContext(schemes=["bcrypt"], rounds=12) password_hash = pwd_context.hash(password) # SECRET_KEY desde .env con 256 bits de entropía
A03
Injection (SQL · NoSQL · Command)
Datos de usuario en queries sin sanitizar
CRÍTICO

El clásico. Input del usuario se interpreta como código. En SQL: '; DROP TABLE users; --. Con SQLAlchemy ORM estás protegido por defecto, pero text() queries son peligrosos.

Payload clásico: ' OR '1'='1
En un login vulnerable: accede sin contraseña

# ❌ VULNERABLE — SQL injection @router.get("/search") async def search(q: str, db=Depends(get_db)): query = f"SELECT * FROM users WHERE name='{q}'" return await db.execute(text(query)) # ☠️ # ✅ SEGURO — ORM con parámetros @router.get("/search") async def search(q: str, db=Depends(get_db)): # SQLAlchemy ORM — parámetros escapados automáticamente result = await db.execute( select(User).where(User.name == q) # ✓ seguro ) return result.scalars().all()
A07
Identification & Authentication Failures
Autenticación rota — tokens, sesiones, contraseñas débiles
ALTO

JWT sin expiración, endpoints sin autenticación requerida, aceptar el algoritmo "none" en JWT, no revocar tokens al logout, no limitar intentos de login.

Un JWT sin exp puede usarse para siempre.
Si se filtra: acceso permanente hasta cambiar SECRET_KEY.

# ❌ JWT sin expiración payload = {"sub": user.id} # sin "exp"! token = jwt.encode(payload, SECRET, "HS256") # ❌ Aceptar cualquier algoritmo jwt.decode(token, SECRET, algorithms=["*"]) # ✅ SEGURO payload = { "sub": str(user.id), "exp": datetime.utcnow() + timedelta(minutes=30), "iat": datetime.utcnow(), "jti": str(uuid4()), # unique token ID } jwt.decode(token, SECRET, algorithms=["HS256"]) # explícito
A05
Security Misconfiguration
Docs expuestas, debug mode, CORS abierto
ALTO

FastAPI genera /docs y /redoc automáticamente — perfectos para desarrollo, peligrosos en producción. CORS con allow_origins=["*"] permite cualquier dominio hacer requests autenticadas.

Un atacante busca /docs en producción primero.
Encuentra todos tus endpoints documentados y los prueba.

# ❌ VULNERABLE — producción con docs app = FastAPI() # docs_url="/docs" por defecto add_middleware(CORSMiddleware, allow_origins=["*"]) # ✅ SEGURO — producción import os IS_PROD = os.getenv("ENV") == "production" app = FastAPI( docs_url=None if IS_PROD else "/docs", redoc_url=None if IS_PROD else "/redoc", openapi_url=None if IS_PROD else "/openapi.json", ) add_middleware(CORSMiddleware, allow_origins=["https://tudominio.com"]) # específico

# tests/security/test_auth.py import pytest from httpx import AsyncClient from jose import jwt @pytest.mark.asyncio async def test_login_brute_force_protection(client: AsyncClient): """Después de N intentos fallidos, debe bloquear.""" for i in range(10): response = await client.post("/api/v1/auth/login", data={"username": "admin@test.com", "password": "wrong"}) # El 11vo intento debe ser bloqueado response = await client.post("/api/v1/auth/login", data={"username": "admin@test.com", "password": "wrong"}) assert response.status_code == 429 # Too Many Requests @pytest.mark.asyncio async def test_jwt_algorithm_none_rejected(client: AsyncClient): """El servidor no debe aceptar JWT con alg:none.""" malicious_token = jwt.encode( {"sub": "1", "role": "admin"}, "", algorithm="none" # ← ataque ) response = await client.get( "/api/v1/users", headers={"Authorization": f"Bearer {malicious_token}"} ) assert response.status_code == 401 # debe rechazar @pytest.mark.asyncio async def test_expired_token_rejected(client: AsyncClient): """Token expirado no debe funcionar.""" from datetime import datetime, timedelta expired_token = jwt.encode( {"sub": "1", "exp": datetime.utcnow() - timedelta(hours=1)}, settings.SECRET_KEY, algorithm="HS256" ) response = await client.get("/api/v1/users", headers={"Authorization": f"Bearer {expired_token}"}) assert response.status_code == 401
# tests/security/test_idor.py import pytest from httpx import AsyncClient @pytest.mark.asyncio async def test_idor_user_cannot_access_other_user(client, user_a_token, user_b_id): """Usuario A no puede ver datos del usuario B.""" response = await client.get( f"/api/v1/users/{user_b_id}", headers={"Authorization": f"Bearer {user_a_token}"} ) assert response.status_code == 403 # Forbidden @pytest.mark.asyncio async def test_idor_user_can_access_own_profile(client, user_a_token, user_a_id): """Usuario A SÍ puede ver su propio perfil.""" response = await client.get( f"/api/v1/users/{user_a_id}", headers={"Authorization": f"Bearer {user_a_token}"} ) assert response.status_code == 200 @pytest.mark.asyncio async def test_idor_sequential_id_enumeration(client, attacker_token): """Enumerar IDs secuenciales no debe revelar otros usuarios.""" success_count = 0 for user_id in range(1, 50): r = await client.get(f"/api/v1/users/{user_id}", headers={"Authorization": f"Bearer {attacker_token}"}) if r.status_code == 200: success_count += 1 assert success_count == 1 # Solo su propio perfil # Correr: pytest tests/security/ -v --tb=short
# app/middleware/rate_limit.py # pip install slowapi from slowapi import Limiter, _rate_limit_exceeded_handler from slowapi.util import get_remote_address from slowapi.errors import RateLimitExceeded from fastapi import FastAPI, Request limiter = Limiter(key_func=get_remote_address) app = FastAPI() app.state.limiter = limiter app.add_exception_handler(RateLimitExceeded, _rate_limit_exceeded_handler) # Proteger endpoints críticos @app.post("/api/v1/auth/login") @limiter.limit("5/minute") # máx 5 intentos por minuto por IP async def login(request: Request, credentials: OAuth2PasswordRequestForm): return await auth_service.login(credentials) @app.post("/api/v1/auth/register") @limiter.limit("3/hour") # anti-spam registration async def register(request: Request, user: UserCreate): return await user_service.create_user(user) # Test en pytest: async def test_rate_limit_login(client): for _ in range(5): await client.post("/api/v1/auth/login", ...) r = await client.post("/api/v1/auth/login", ...) assert r.status_code == 429 # Too Many Requests ✓
# tests/security/test_headers.py # Verificar que los headers de seguridad están presentes import pytest REQUIRED_SECURITY_HEADERS = { "X-Frame-Options": "DENY", "X-Content-Type-Options": "nosniff", "Strict-Transport-Security": lambda v: "max-age" in v, "Content-Security-Policy": lambda v: "default-src" in v, "X-Powered-By": "", # debe estar vacío } @pytest.mark.asyncio async def test_security_headers_present(client): response = await client.get("/") for header, expected in REQUIRED_SECURITY_HEADERS.items(): actual = response.headers.get(header, "") if callable(expected): assert expected(actual), f"Header {header} inválido: {actual}" else: assert actual == expected, f"Header {header}: {actual} != {expected}" @pytest.mark.asyncio async def test_server_header_hidden(client): """El servidor no debe revelar tecnología en el header Server.""" response = await client.get("/") server = response.headers.get("server", "").lower() assert "uvicorn" not in server # no revelar stack assert "python" not in server # Ejecutar todo: # pytest tests/security/ -v --cov=app --cov-report=html

// sección 08 — mentalidad

El mindset
del hacker ético

"Un hacker ético piensa en todos los casos extremos que un desarrollador normal ignoraría. No es paranoia — es rigor."
— Principio del Red Team
01

Piensa como el atacante

Cada endpoint que creas, pregúntate: ¿qué pasa si el input es '; DROP TABLE? ¿si el user_id es negativo? ¿si el archivo tiene extensión falsa? El pensamiento adversarial es una habilidad que se entrena.

02

Nunca asumas confianza

Todo input del exterior es potencialmente malicioso — URL params, headers, body JSON, archivos. Pydantic valida estructura pero no intención. Valida el negocio también, no solo el tipo.

03

Defensa en profundidad

Ninguna capa de seguridad es perfecta sola. TLS puede fallar, el WAF puede eludirse, el JWT puede comprometerse. El objetivo es que el atacante tenga que romper 5 capas, no 1.

04

Privilege of least access

Tu API no necesita permisos de superusuario en PostgreSQL. El proceso de FastAPI no necesita acceso root en el servidor. Cada componente debe tener exactamente los permisos que necesita — ni uno más.

ENTRADA #004 — CAPÍTULO 04 COMPLETADO

Hoy entendí que el hacking ético no es técnica — es perspectiva. La misma FastAPI que construí para servir usuarios, vista desde el lado de un atacante, es una superficie de ataque con 10 vectores diferentes. IDOR me pareció el más revelador: una línea de código faltante (if current_user.id != user_id) y todos los datos de todos los usuarios están expuestos.

El siguiente paso concreto: instalar Kali Linux en WSL2, correr crAPI en Docker localmente, y completar el path "Jr Penetration Tester" en TryHackMe. Los security tests con pytest van al CI/CD desde hoy — cada push valida que la API no tiene IDOR, que rate limiting funciona, y que los headers de seguridad están presentes.
